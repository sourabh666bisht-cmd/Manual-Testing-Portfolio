
# Test Cases – SauceDemo Manual Testing Project

## Overview

This folder contains comprehensive test cases designed for the SauceDemo E-Commerce web application.

The objective of these test cases is to validate the application's functionality, usability, security, accessibility, and overall user experience from an end-user perspective.

---

## Scope Covered

The following modules were tested:

### Authentication
- Login with valid credentials
- Login with invalid credentials
- Locked user validation
- Session management
- Logout functionality

### Inventory Management
- Product listing validation
- Product details validation
- Product sorting
- Product image verification
- Price validation

### Cart Management
- Add to Cart
- Remove from Cart
- Cart badge validation
- Cart persistence validation

### Checkout Process
- Customer information validation
- Mandatory field validation
- Order overview validation
- Tax and total calculation verification
- Order completion flow

### UI / UX Testing
- Layout consistency
- Responsive design validation
- Content verification
- Navigation testing

### Accessibility Testing
- Keyboard navigation
- Tab order validation
- Focus visibility checks
- Screen reader compatibility checks

### Security Testing
- Input validation
- Session handling
- URL manipulation checks
- Authorization validation

---

## Test Design Techniques Used

- Positive Testing
- Negative Testing
- Boundary Value Analysis (BVA)
- Equivalence Partitioning
- Exploratory Testing
- Error Guessing
- Risk-Based Testing
