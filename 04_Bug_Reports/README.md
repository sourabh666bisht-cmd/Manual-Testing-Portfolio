# Bug Reports – SauceDemo Manual Testing Project

## Overview

This folder contains defects identified during manual testing of the SauceDemo E-Commerce application.

The defects were documented using standard defect reporting practices followed in software testing projects.

Each defect includes detailed information such as reproduction steps, expected behavior, actual behavior, severity, priority, environment details, and current status.

---

## Defect Reporting Process

The following workflow was followed:

1. Defect Identification
2. Defect Logging
3. Severity & Priority Assignment
4. Defect Validation
5. Retesting
6. Closure Verification

---

## Defect Attributes Captured

Each bug report contains:

* Bug ID
* Module
* Summary
* Description
* Preconditions
* Steps to Reproduce
* Expected Result
* Actual Result
* Severity
* Priority
* Environment
* Status

---

## Severity Classification

| Severity | Description                                                |
| -------- | ---------------------------------------------------------- |
| Critical | Application crash, data loss, or business-critical failure |
| High     | Major functionality impacted with no workaround            |
| Medium   | Functionality partially impacted                           |
| Low      | Minor issue with limited business impact                   |

---

## Priority Classification

| Priority | Description                             |
| -------- | --------------------------------------- |
| High     | Requires immediate fix                  |
| Medium   | Fix planned in upcoming release         |
| Low      | Can be fixed later without major impact |
