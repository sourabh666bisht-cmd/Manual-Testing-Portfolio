# Requirement Traceability Matrix (RTM) – SauceDemo Manual Testing Project

## Overview

This folder contains the Requirement Traceability Matrix (RTM) for the SauceDemo E-Commerce application.

The RTM ensures that all business and functional requirements are properly mapped to test cases and that no requirement remains untested during the testing lifecycle.

---

## Purpose of RTM

The primary objectives of RTM are:

* Ensure complete requirement coverage
* Verify that every requirement has corresponding test cases
* Identify missing test coverage
* Support impact analysis during requirement changes
* Improve test planning and reporting

---

## Requirement Coverage Process

The following approach was used:

1. Analyze Business Requirements
2. Identify Functional Requirements
3. Create Test Scenarios
4. Design Test Cases
5. Map Requirements to Test Cases
6. Validate Coverage
7. Track Execution Status

## RTM Structure

The RTM document contains:

| Field                   | Description                   |
| ----------------------- | ----------------------------- |
| Requirement ID          | Unique requirement identifier |
| Requirement Description | Requirement details           |
| Priority                | Business priority             |
| Test Case ID            | Linked test cases             |
| Coverage Status         | Covered / Not Covered         |
| Remarks                 | Additional notes              |
