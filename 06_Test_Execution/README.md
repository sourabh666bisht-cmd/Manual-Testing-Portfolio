# Test Execution – SauceDemo Manual Testing Project

## Overview

This folder contains the Test Execution Report for the SauceDemo E-Commerce application.

The objective of test execution is to validate the application against documented test cases and record the execution status of each test scenario.

---

## Test Execution Approach

The following execution process was followed:

1. Review Test Cases
2. Prepare Test Environment
3. Execute Test Cases
4. Record Results
5. Log Defects
6. Retest Fixed Defects
7. Update Execution Status
8. Generate Execution Summary

---

## Execution Status Categories

| Status       | Description                                      |
| ------------ | ------------------------------------------------ |
| Pass         | Test case executed successfully                  |
| Fail         | Actual result differs from expected result       |
| Blocked      | Execution cannot proceed due to dependency issue |
| Not Executed | Test case pending execution                      |
